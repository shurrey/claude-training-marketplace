# AI-DLC — common discipline

These are cross-cutting rules that apply across every AI-DLC phase. They cover
*how* to interact during AI-DLC work — independent of which phase you're in.

Key concepts to apply:

- **Depth levels** — calibrate how deep to go based on user signals (see `references/depth-levels.md`)
- **Overconfidence prevention** — explicitly check assumptions and ask before guessing (see `references/overconfidence-prevention.md`)
- **Content validation** — verify generated content against stated requirements (see `references/content-validation.md`)
- **Question format** — structure clarifying questions consistently (see `references/question-format-guide.md`)
- **Session continuity** — recover state across interrupted sessions via `aidlc-docs/aidlc-state.md` (see `references/session-continuity.md`)
- **ASCII diagram standards** — for any diagrams you draft (see `references/ascii-diagram-standards.md`)
- **Welcome message** — initial framing when AI-DLC starts (see `references/welcome-message.md`)
- **Workflow changes** — how to handle in-flight scope changes (see `references/workflow-changes.md`)
